  1. **`scripts/local_checks.py` is an ADR-035-governed file** (ADR-035 §1 names it explicitly). Any change to it must also update Makefile targets (ADR-035 §3 local reproducibility parity). A `make governance-status` target is therefore required alongside the `local_checks.py` step addition.
  2. **The manual CI wiring step** (adding the producer call and `upload-artifact` action to `.github/workflows/ci.yml`) touches an ADR-035-governed path. 1. **`telemetry_summary` with event counts is unachievable locally** — no governance event count fixture reports exist. Replaced with `schema_checks` section aggregating `ok` status from existing quality report files.
2. **`run.*` env vars are CI-only** — producer uses `GITHUB_WORKFLOW`, `GITHUB_RUN_ID`, `GITHUB_SHA` with explicit `null` fallbacks when not in CI environment.
3. **CI wiring requires `.github/workflows/ci.yml` edit** — cannot be automated from local tooling; documented as a manual step in artifact docs.

5. **ADR-035 not referenced in original plan** — added to governing references.
6. **Config manager report uses `total_violations` not `ok`** — producer handles both report formats.
7. **Schema must have `additionalProperties: false`** — required for ADR-028 governance hardening consistency with existing schemas.
