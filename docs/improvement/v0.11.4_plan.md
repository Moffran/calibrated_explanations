## Task 2: ADR-004 Gap 1 â€” Deprecate `strategy="auto"` when `enabled=True`

### 1. Open questions

None. ADR-004 Â§Decision explicitly prohibits automatic backend selection. The deprecation path is clear.

### 2. Goal

Emit a `deprecate()` call (via `utils/deprecations.py`) when a caller sets `enabled=True` without an explicit strategy (or with `strategy="auto"`). This gives users the two-minor deprecation window required by ADR-011 before v1.0.0 removes `strategy="auto"` entirely.

### 3. Why this task exists

ADR-004 Â§Decision: "no automatic strategy selection." `ParallelConfig(strategy="auto")` is the current default when `enabled=True`, and the `auto_strategy()` heuristic selects a backend based on CPU count, CI detection, OS, and task size â€” making behavior non-deterministic across machines. The deprecation must land in v0.11.4 (or earlier) so that v1.0.0 can enforce explicit strategy without violating the two-minor window.

### 4. Governing ADRs

- `docs/improvement/adrs/ADR-004-parallel-execution-framework.md`
- `docs/improvement/adrs/ADR-011-deprecation-and-migration-policy.md` (Â§2 two-minor window)

### 5. Hardening / red-team findings

Appendix Gap 1 (severity 9): `ParallelConfig(strategy="auto")` at `parallel/parallel.py:517-589`. The `auto_strategy()` function is called when `strategy` resolves to `"auto"`. The deprecation must fire when `enabled=True AND strategy == "auto"` â€” not when `enabled=False` (no backend is selected at all).

### 6. Compatibility posture

Deprecation only; no removal. Code that does not set `enabled=True` is unaffected. Code that sets `enabled=True` with an explicit strategy (not `"auto"`) is unaffected. Only code that relies on automatic backend selection is warned.

## Task 6: ADR-008 Gap 3 â€” Structured calibration/model metadata on `Explanation` dataclass

### 1. Open questions

**Schema question:** Should the new fields be typed or remain generic? Options:
- **Option A (typed):** Add `CalibrationDescriptor` and `ModelDescriptor` dataclasses and typed fields on `Explanation`.
- **Option B (structured dict):** Add `calibration_metadata: Optional[Dict[str, Any]]` and `model_metadata: Optional[Dict[str, Any]]` with documented key conventions.

Option B is strongly preferred for v0.11.4: it is additive, backward-compatible, and does not require a new public contract (the domain model is internal). Option A requires a new public API surface, which belongs in v1.0.0-rc design work. Confirm with user.
