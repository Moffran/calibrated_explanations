## Decision

PlotSpec is promoted to the default user-facing plotting path for all in-scope
plotting surfaces:

- `plot_probabilistic`
- `plot_regression`
- `plot_alternative`
- `plot_triangular`
- `plot_global`
- factual explanation `.plot()`
- alternative explanation `.plot()`
- batch/collection explanation `.plot()` entrypoints that route to the plotting
  functions above

Legacy rendering remains available only as an explicit opt-out or visible
fallback path.

## Compatibility and fallback posture

Supported explicit legacy opt-out paths:

- `use_legacy=True`
- `style_override="legacy"` where the plotting function accepts style overrides
- explicit legacy style selection through supported plugin/style configuration

When PlotSpec rendering fails and execution falls back to legacy, the fallback
must emit both:

- `UserWarning`
- INFO log entry

No deprecation warning is required for the default flip itself because no public
symbol is removed by this task.

## ADR and standards conformance

- ADR-036: PlotSpec remains the canonical semantic IR on promoted paths; dict/JSON
  forms are boundary serialization artifacts only.
- ADR-037: Runtime plot-kind extension remains disabled; this task does not
  introduce new extension registration semantics.
- ADR-011: This task does not remove public symbols. Remaining legacy module
  removal belongs to Task 1.
- ADR-030 and STD-003: Default-routing tests assert public behavior and legacy
  opt-out behavior.
- STD-004: `docs/api/visualization.md` documents the default path, opt-out path,
  and fallback behavior at reference depth.
- STD-005: Fallback behavior remains visible through warning and INFO logging.

## Bridge A implication

Task 6 removes default-path reliance on `legacy/plotting.py`, but explicit opt-out
and fallback imports may remain until Task 1 removes Bridge A. Task 1 must still
verify and migrate any remaining legacy imports before deleting
`src/calibrated_explanations/legacy/plotting.py`.
