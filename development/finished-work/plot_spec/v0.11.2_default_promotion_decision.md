## Decision

The binding v0.11.2 outcome is the **1A-family deferral decision**:

- Keep legacy plotting as the default in v0.11.2.
- Defer chain-based default promotion for `plot_probabilistic`, `plot_regression`, and `plot_alternative` to v0.11.3.
- Defer explicit default-promotion decisions for `plot_triangular` and `plot_global` to v0.11.3.

Gate readiness evidence is recorded, but it is **not sufficient to authorize promotion** in v0.11.2.

## Deferral rationale

The current blocker is **not** canonical PlotSpec payload correctness alone. The blocker is that the current PlotSpec-rendered visuals are not yet trusted to preserve the same explanatory meaning that legacy plots communicate to users.

In other words:

- PlotSpec payload and routing readiness are necessary.
- Meaning-preserving visual rendering is also necessary.
- v0.11.2 has the first two far enough along to continue improving the opt-in path, but not far enough to change the default path safely.

## Current opt-in / opt-out surface

PlotSpec remains available in v0.11.2 on explicit opt-in paths:

- `use_legacy=False`
- `style_override="plot_spec.default"`
- `return_plot_spec=True`
- project/runtime configuration that explicitly prefers `plot_spec.default`

Legacy remains the default path in v0.11.2:

- public explanation plotting defaults remain legacy-oriented
- bypass functions `plot_triangular` and `plot_global` continue to default to legacy behavior
