## Scope

## Red-team findings (2026-04-24 subagent + direct code verification)

A subagent red-team was run and then its findings were verified by direct source inspection.
Several "CRITICAL" claims were incorrect; the corrected record is:

**Claims that turned out to be false (no mending required):**
1. "Regression bar colors all red" — **FALSE**. Adapter line 1610: `color = "b" if width > 0 else "r"` matches legacy.
2. "Triangular rendering stubbed" — **FALSE**. `_render_triangular_spec` is a complete matplotlib implementation with triangle reference lines, quiver arrows, and scatter.
3. "Global rendering has no PlotSpec pipeline" — **FALSE**. `_render_global_spec` is fully implemented with class-stratified scatter and threshold-derived labels.
4. "neg_caption always renders P(y=X)" — **FALSE**. `plotting.py` lines 702–763 compute `neg_caption = "P(y=0)"` or `"P(y!={label})"` and pass them explicitly.
5. "Alternative probabilistic header always absent" — **FALSE**. For classification `plotting.py` always sets `xticks_override` which triggers `header_needed=True` in the builder.

**Claims that were confirmed real:**
6. Semantic parity tests were shallow (structure checks only). New tests were added and the parity surface is now broader than the original route-only checks.
7. Conjunction-style multiline label legibility was not explicitly verified even though conjunction artifacts are generated as part of the parity packet.

**Prior red-team findings still valid:**
8. `GlobalSpec` carries `threshold`, `class_labels`, and `is_regularized` as semantic fields to support label derivation in the adapter.
9. Boundary serialization normalizes numpy scalars and arrays.
